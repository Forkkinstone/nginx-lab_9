Тема: Настройка непрерывной интеграции (CI) с использованием GitHub Actions.

1. Цель работы
Автоматизация процесса тестирования веб-приложения при внесении изменений в репозиторий с использованием технологий CI/CD и Docker Compose.

2. Ход работы
2.1. Подготовка окружения
Проект был перенесен в новую рабочую область web-nginx_9. Были обновлены конфигурации docker-compose.yml для обеспечения изоляции контейнеров (использование префикса lab9_). Проведена локальная проверка работоспособности тестов.

2.2. Разработка сценария CI
В директории .github/workflows/ был создан файл конфигурации main.yml, описывающий этапы автоматической сборки:

Checkout: Получение актуального кода из репозитория.

Start containers: Поднятие инфраструктуры (PHP, Nginx, RabbitMQ) через docker compose.

Install Dependencies: Установка библиотек через Composer внутри контейнера.

Run Tests: Автоматический запуск PHPUnit тестов.

2.3. Отладка пайплайна
В процессе настройки были устранены следующие ошибки:

Exit code 127: Исправлен синтаксис команды (переход с docker-compose на docker compose).

Exit code 126: Решена проблема прав доступа к исполняемым файлам (запуск тестов через интерпретатор php).

Race condition: Добавлена задержка sleep 10 для обеспечения готовности сервисов (RabbitMQ) перед началом тестирования.

3. Результаты
Сценарий успешно запускается при каждом пуше в ветку main. Все тесты проходят в автоматическом режиме в облачной инфраструктуре GitHub.

4. Вывод
Настройка CI/CD позволяет значительно сократить количество ошибок при интеграции нового кода. Использование GitHub Actions в связке с Docker гарантирует, что тесты запускаются в идентичном окружении как у разработчика, так и на сервере, что является стандартом современной промышленной разработки.
